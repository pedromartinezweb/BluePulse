# Security Policy

## Supported versions

BluePulse is early-stage software. Security fixes target the latest source on `main` and the latest signed release.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Report security concerns by email:

```text
info@pedromartinezweb.com
```

Include:

- A clear description of the issue.
- Steps to reproduce it.
- The affected version or commit.
- Any relevant logs, screenshots, or proof of concept.

## Project boundaries

BluePulse must not:

- Simulate keyboard or mouse input.
- Install background services.
- Bypass corporate lock policies.
- Collect user data.
- Add network behavior without a documented security review.
